# Plan 02-01: Explorer Tree/List Views

## Objective

Create explorer components for browsing the active workspace in both tree and list views, backed by indexed SQLite data.

## Current State

- Database has `list_children` function to query files by parent path
- App shell exists with basic ExplorerView component (list only)
- No tree view implementation yet

## Implementation Tasks

### Task 1: Create ExplorerList Component

Create a dedicated list view component that displays files/folders for the current directory.

**Files to create:**
- `frontend/src/components/ExplorerList.tsx`

**Behavior:**
- Display current path breadcrumb
- Show ".." entry to navigate to parent (if not at root)
- List children with: icon, name, size (files only), modified date
- Single click: select item
- Double click: open folder or select file
- Keyboard navigation: arrows, enter, backspace

**Props interface:**
```typescript
interface ExplorerListProps {
  currentPath: string;
  rootPath: string;
  items: FileRecord[];
  selectedPath?: string;
  onSelect: (record: FileRecord) => void;
  onNavigate: (path: string) => void;
  onOpenParent: () => void;
}
```

### Task 2: Create ExplorerTree Component

Create a tree view component for hierarchical browsing.

**Files to create:**
- `frontend/src/components/ExplorerTree.tsx`

**Behavior:**
- Display root folder initially
- Expandable folders with chevron indicator
- Lazy-load children when expanding
- Show folder icon for directories, file icon for files
- Single click: select item
- Double click: expand/collapse folder
- Maintain expansion state

**Props interface:**
```typescript
interface ExplorerTreeProps {
  rootPath: string;
  selectedPath?: string;
  onSelect: (record: FileRecord) => void;
  onNavigate: (path: string) => void;
}
```

### Task 3: Add View Mode Switcher

Add UI to switch between tree and list views in explorer mode.

**Files to modify:**
- `frontend/src/main.tsx`

**Behavior:**
- Toggle buttons for Tree/List view
- Remember last used view per session
- Default to list view

### Task 4: Backend Support for Tree Data

Ensure backend supports efficient tree queries.

**Files to verify:**
- `src-tauri/src/db.rs` - `list_children` function exists
- `src-tauri/src/main.rs` - `list_children` command registered

**Verification:**
- Confirm `list_children` returns correct parent-child relationships
- Test with deeply nested directories
