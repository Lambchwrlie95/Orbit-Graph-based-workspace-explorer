# Plan 02-02: Selection State, Inspector, and Preview

## Objective

Implement shared selection state, inspector panel with metadata and actions, and basic preview for supported file types.

## Current State

- Basic selection state exists in App component
- Inspector component exists but is minimal
- Preview component exists but is basic
- External open command exists

## Implementation Tasks

### Task 1: Enhance Selection State Management

Improve selection state to work across all views.

**Files to modify:**
- `frontend/src/main.tsx`
- `frontend/src/hooks/useSelection.ts` (create)

**Behavior:**
- Selection persists when switching views
- Selection updates inspector immediately
- Clear selection when changing root workspace

### Task 2: Enhance Inspector Component

Expand inspector to show full metadata and actions.

**Files to modify:**
- `frontend/src/components/Inspector.tsx`

**Display:**
- File icon and name
- Full path (truncated with tooltip)
- Type (folder, mime type, or extension)
- Size (human readable)
- Created date
- Modified date
- Extension

**Actions:**
- Open externally (primary action)
- Copy path to clipboard
- Show in parent folder (navigate explorer to parent)

### Task 3: Enhance Preview Component

Improve preview for text and image files.

**Files to modify:**
- `frontend/src/components/Preview.tsx`
- `src-tauri/src/preview.rs`

**Text file preview:**
- Show first 100 lines or 50KB (whichever is smaller)
- Syntax highlighting for common extensions (optional)
- Line numbers (optional)
- Scrollable

**Image preview:**
- Display image at reasonable size
- Show dimensions in metadata
- Support: png, jpg, jpeg, gif, svg, webp, bmp, ico

**File types to support:**
- Text: .txt, .md, .rs, .ts, .tsx, .js, .json, .toml, .yaml, .yml, .css, .html
- Images: .png, .jpg, .jpeg, .gif, .svg, .webp, .bmp, .ico

### Task 4: Backend Preview Enhancement

Update Rust preview module for better text and image handling.

**Files to modify:**
- `src-tauri/src/preview.rs`

**Text preview:**
- Read first N bytes only
- Detect binary files and reject
- Return truncated indicator if file is larger

**Image preview:**
- Load image dimensions
- Return base64 data URL for images
- Handle errors gracefully

### Task 5: Open Externally Integration

Wire up external open throughout the UI.

**Files to verify:**
- `src-tauri/src/main.rs` - `open_path` command exists
- `frontend/src/components/Inspector.tsx` - Open button works

**Add to:**
- Explorer list: double-click files to open
- Explorer tree: double-click files to open
- Search results: double-click to open

## Verification Steps

1. Select a file in explorer
2. Verify inspector shows all metadata
3. Click "Open" and verify file opens externally
4. Select a text file and verify preview shows content
5. Select an image file and verify preview shows image
6. Select a folder and verify different actions available
7. Copy path and verify clipboard has correct path

## Done Criteria

- [ ] Selection state shared across all views
- [ ] Inspector shows full metadata for selected item
- [ ] Inspector actions work (open, copy path)
- [ ] Text files show preview with content
- [ ] Image files show preview with image
- [ ] External open works from all entry points
