## Technical Context

### Desktop Entry Specification
- Reference: https://specifications.freedesktop.org/desktop-entry-spec/
- Desktop files must be in `/usr/share/applications/` (system) or `~/.local/share/applications/` (user)
- Icon names are resolved via XDG icon theme specification

### Tauri Icon Requirements
- Tauri expects icons in `src-tauri/icons/`
- Default icon names: 32x32.png, 128x128.png, 128x128@2x.png, icon.icns, icon.ico
- For Linux: PNG files are used directly

### Icon Size Requirements
- 16x16: Menu items, small toolbars
- 32x32: Standard toolbar size
- 48x48: Standard application icon
- 128x128: High DPI displays
- 256x256: Large displays, app stores
- 512x512: Maximum quality source
- SVG: Scalable, preferred by modern desktops

<task type="auto">
  <name>Task 2: Create main desktop entry file</name>
  <files>src-tauri/orbit.desktop</files>
  <action>
Create the primary desktop entry file for Orbit. This file will be installed to the system applications directory and appears in the main application menu.

**File: src-tauri/orbit.desktop**

```ini
[Desktop Entry]
Name=Orbit
Comment=Graph-first file intelligence IDE
Exec=orbit %F
Icon=orbit
Type=Application
Terminal=false
Categories=System;FileTools;FileManager;Development;IDE;
StartupNotify=true
StartupWMClass=orbit
MimeType=inode/directory;
Keywords=file;manager;graph;ide;code;explorer;
```

**Field explanations:**
- `Name`: Display name in menus
- `Exec`: Command to launch; %F allows passing files/folders
- `Icon`: Icon name (searched in icon paths); use "orbit"
- `Categories`: Determines menu placement (System → File Tools)
- `MimeType`: Declares ability to open directories (for right-click)
- `StartupWMClass`: Links window to desktop entry for taskbar grouping
- `Keywords`: Search terms for application launchers

**Note:** The `Exec` path will be resolved by the package installer. For AppImage, Tauri handles this automatically.
  </action>

  </verify>

<task type="auto">
  <name>Task 3: Create folder quick-launcher desktop entry</name>
  <files>src-tauri/orbit-folder.desktop</files>
  <action>
Create a secondary desktop entry for the "Open Folder in Orbit" context menu action. This entry will NOT appear in the main application menu (NoDisplay=true).

**File: src-tauri/orbit-folder.desktop**

```ini
[Desktop Entry]
Name=Open Folder in Orbit
Comment=Open selected folder in Orbit file intelligence IDE
Exec=orbit --folder %f
Icon=orbit
Type=Application
Terminal=false
Categories=System;FileTools;FileManager;Development;
MimeType=inode/directory;
NoDisplay=true
OnlyShowIn=GNOME;KDE;XFCE;MATE;Cinnamon;Budgie;LXQt;Unity;
```

**Why a separate entry:**
- `NoDisplay=true` hides it from main menus
- The specific `Exec` command opens a folder directly
- Can be registered specifically for inode/directory MIME type
- Provides clean separation of use cases

**Right-click integration:**
When a user right-clicks a folder, the file manager checks desktop entries with matching MimeType. This entry will appear as "Open Folder in Orbit" in the context menu.

<task type="auto">
  <name>Task 5: Create icon generation script for future updates</name>
  <files>scripts/generate-icons.sh</files>
  <action>
Create a helper script to regenerate icons from a source image. This makes it easy to update the icon design later.

**File: scripts/generate-icons.sh**

```bash
#!/bin/bash
# Generate Orbit icons at all required sizes
# Usage: ./scripts/generate-icons.sh [source-icon.png]

set -e

SOURCE="${1:-src-tauri/icons/icon.png}"
OUTPUT_DIR="src-tauri/icons"

echo "Generating Orbit icons from: $SOURCE"
echo "Output directory: $OUTPUT_DIR"

# Check if source exists
if [ ! -f "$SOURCE" ]; then
    echo "Error: Source icon not found: $SOURCE"
    exit 1
fi

# Check for ImageMagick
if ! command -v convert &> /dev/null; then
    echo "Error: ImageMagick 'convert' not found. Install with:"
    echo "  Ubuntu/Debian: sudo apt-get install imagemagick"
    echo "  Fedora: sudo dnf install imagemagick"
    exit 1
fi

# Generate sizes
echo "Generating 32x32..."
convert "$SOURCE" -resize 32x32 "$OUTPUT_DIR/32x32.png"

echo "Generating 128x128..."
convert "$SOURCE" -resize 128x128 "$OUTPUT_DIR/128x128.png"

echo "Generating 128x128@2x..."
convert "$SOURCE" -resize 256x256 "$OUTPUT_DIR/128x128@2x.png"

echo "Generating 256x256..."
convert "$SOURCE" -resize 256x256 "$OUTPUT_DIR/256x256.png"

echo "Generating 512x512..."
convert "$SOURCE" -resize 512x512 "$OUTPUT_DIR/512x512.png"

# Ensure icon.png is 128x128 (Tauri default)
if [ "$SOURCE" != "$OUTPUT_DIR/icon.png" ]; then
    echo "Copying to icon.png..."
    cp "$OUTPUT_DIR/128x128.png" "$OUTPUT_DIR/icon.png"
fi

echo ""
echo "Icon generation complete!"
echo "Generated files:"
ls -la "$OUTPUT_DIR"/*.png
```

**Make it executable:**
```bash
chmod +x scripts/generate-icons.sh
```

This script provides a reproducible way to regenerate icons and documents the requirements.
