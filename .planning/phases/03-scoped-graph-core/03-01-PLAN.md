# 03-01: Graph API, Visible-Scope Query, and Graphology Data Loading

## Objective

Implement the backend Graph API to query scoped graph data from SQLite and load it into Graphology in the frontend.

## Context

The graph must be explicitly scoped by root path and current scope path. The backend returns nodes, edges, and metadata about the scope (total count, capping status). The frontend loads this into a Graphology graph for Sigma.js rendering.

## Implementation Details

### Node Query Strategy

Nodes are queried with folder-first, depth-first ordering:
1. Root scope folder first
2. Direct children next
3. Then progressively deeper levels

This ensures important nodes are included even when capped.

### Folder Clustering Algorithm

1. Count children per parent in the result set
2. For folders with >50 visible children:
   - Show first 20 children
   - Create a "+N more" cluster node
   - Hide remaining children from the graph

### Edge Generation

Two sources of edges:
1. **Synthetic containment edges**: Generated from parent-child relationships
2. **Database edges**: From the `edges` table for explicit relationships
