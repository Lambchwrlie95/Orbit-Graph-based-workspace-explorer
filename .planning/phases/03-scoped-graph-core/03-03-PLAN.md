# 03-03: Node Caps, Folder Clustering, and Graph-to-Inspector/Open Integration

## Objective

Enforce node limits to prevent graph overload, implement intelligent folder clustering, and fully integrate graph interactions with the rest of the application.

## Context

Large folders can overwhelm the graph visualization. We need to cap visible nodes at 200 and intelligently cluster folders that exceed 50 children. The graph must integrate seamlessly with the inspector panel and workspace navigation.

## Tasks Completed

### Task 1: Enforced Node Cap (200 nodes)

**Backend (`src-tauri/src/graph.rs`):**
- Set `DEFAULT_NODE_LIMIT` to 200
- Queries limited to 200 nodes maximum
- Returns `capped` flag and `totalInScope` count
- Shows capping status in UI overlay

**Frontend (`frontend/src/main.tsx`):**
- Updated `loadGraph` to request 200 nodes
- Status bar shows capping information

### Task 2: Folder Clustering

**Algorithm implemented:**
1. Detect folders with >50 visible children
2. Create cluster node labeled "+N more"
3. Show only first 20 children
4. Hide remaining children from view
5. Connect cluster node to parent folder

**Cluster node attributes:**
- Label: "+N more" (e.g., "+42 more")
- Color: Amber (#f59e0b)
- Size: 18px (largest)
- Type: cluster (boolean flag)

### Task 3: Graph Mode Integration

**Main App (`frontend/src/main.tsx`):**
- Graph mode in mode switcher
- Graph loads when switching to graph mode
- Graph reloads when current path changes
- Scope path passed to loadGraph command

### Task 4: Graph-to-Inspector Flow

**Bidirectional integration:**

**Graph → Inspector:**
- Click node → `handleGraphNodeSelect` → `selectRecord` → Inspector updates
- Cluster node paths handled specially (strip `/__cluster__` suffix)

**Graph → Navigation:**
- Double-click folder → `onFocusFolder` → `setCurrentPath` → Graph reloads
- Double-click file → `onOpenPath` → External open
- Double-click cluster → Navigate to parent folder

### Task 5: Graph Navigation

**Navigation flow:**
1. **Click folder**: Select in inspector only
2. **Double-click folder**: Enter/focus that folder (new scope)
3. **Double-click file**: Open externally
4. **Double-click cluster**: Navigate to parent folder contents
