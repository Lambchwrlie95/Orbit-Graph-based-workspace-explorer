<MillerColumnsPattern>
Miller columns (also called cascading lists) display hierarchical data in multiple panes:
- Each column shows the contents of one folder
- Selecting a folder in column N shows its contents in column N+1
- Users can see multiple levels of hierarchy simultaneously
- Common in macOS Finder, NeXTSTEP, and modern file managers

Key behaviors:
1. Root column shows workspace or current scope root
2. Clicking a folder adds a new column to the right
3. Clicking a file selects it (no new column)
4. Columns scroll horizontally if they overflow
5. Selection in column N removes columns N+2, N+3, etc.
</MillerColumnsPattern>

```typescript
const buildColumnsFromPath = async (fullPath: string) => {
  const newColumns: ColumnState[] = [];
  let current = rootPath;
  
  // Add root column
  const rootItems = await invoke<FileRecord[]>("list_children", { parentPath: rootPath });
  newColumns.push({
    path: rootPath,
    name: rootPath.split("/").pop() || "Root",
    items: rootItems,
    width: 250,
  });
  
  // Add intermediate columns
  const relativePath = fullPath.slice(rootPath.length);
  const segments = relativePath.split("/").filter(Boolean);
  
  for (const segment of segments) {
    current = `${current}/${segment}`;
    const items = await invoke<FileRecord[]>("list_children", { parentPath: current });
    newColumns.push({
      path: current,
      name: segment,
      items,
      width: 250,
    });
  }
  
  setColumns(newColumns);
  setActiveColumnIndex(newColumns.length - 1);
};
```

3. **Selection cascade**:
   - When user selects folder in column N:
     - Remove columns N+1, N+2, etc.
     - Load column N+1 with folder's contents
     - Set active column to N+1
   - When user selects file:
     - Call onSelect(record)
     - Remove columns after current (optional: keep them)

4. **Column selection handler**:
```typescript
const handleColumnSelect = (columnIndex: number, record: FileRecord) => {
  // Update selected path in column
  setColumns(prev => prev.map((col, idx) => 
    idx === columnIndex ? { ...col, selectedPath: record.path } : col
  ));
  
  if (record.isDir) {
    // Load children and add new column
    setIsLoading(true);
    invoke<FileRecord[]>("list_children", { parentPath: record.path })
      .then(items => {
        setColumns(prev => {
          // Remove columns after this one
          const newCols = prev.slice(0, columnIndex + 1);
          newCols.push({
            path: record.path,
            name: record.name,
            items,
            width: 250,
          });
          return newCols;
        });
        setActiveColumnIndex(columnIndex + 1);
        onNavigate(record.path);
      })
      .finally(() => setIsLoading(false));
  } else {
    onSelect(record);
  }
};
```

5. **Global keyboard navigation**:
   - Attach keydown listener to container
   - ArrowLeft: move to previous column (if activeColumnIndex > 0)
   - ArrowRight: move to next column (if activeColumnIndex < columns.length - 1)
   - Focus management: track which column has focus

6. **Horizontal scrolling**:
   - Container uses `overflow-x: auto`
   - Ensure last column is visible when adding new columns
   - Smooth scroll to new columns

7. **Loading states**:
   - Show loading indicator in column being populated
   - Disable interaction during load

8. **Root path handling**:
   - Always show root column
   - ".." navigation disabled at root
