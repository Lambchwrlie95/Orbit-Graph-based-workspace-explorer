## Overview

Orbit v1 builds the real foundation for a graph-first file intelligence IDE. The path starts with a reliable Tauri/Rust/SQLite scanner, then adds practical explorer/search/inspector workflows, then ships a scoped Sigma/Graphology graph, and finally hardens performance safeguards so large local folders do not break the experience.

## Completed Phases

### ✅ Phase 1: Workspace Index Foundation
**Goal**: User can open a folder, scan it, and persist metadata in SQLite.

**Success Criteria**:
1. User can choose a folder and see it as the active workspace.
2. User can start a scan and see scan progress, completion, and errors.
3. SQLite contains file/folder metadata for the scanned workspace.
4. Re-running a scan skips unchanged files where metadata allows it.
5. Scan, SQLite, watcher, and graph-scope issues are written to durable app logs.

**Plans Completed**:
- ✅ 01-01: Tauri 2, React, TypeScript, Rust, and SQLite app shell
- ✅ 01-02: Workspace open command, scan job, metadata extraction, and database schema
- ✅ 01-03: Scan progress, durable logging, error reporting, and unchanged-file skip behavior

**Requirements**: 8/8 (WORK-01 through WORK-03, INDX-01 through INDX-05)

---

### ✅ Phase 2: Explorer, Search, and Inspector
**Goal**: User can browse, search, inspect, preview, and externally open indexed files.

**Success Criteria**:
1. User can browse the active workspace in tree and list views.
2. User can select any indexed item and see metadata plus available actions.
3. User can search by filename and select a result.
4. User can open a selected file externally.
5. Supported text/image files show a basic preview.

**Plans Completed**:
- ✅ 02-01: Explorer tree/list views backed by indexed data
- ✅ 02-02: Shared selection state, inspector metadata, actions, and basic preview
- ✅ 02-03: Filename search and search-result selection/open flows

**Requirements**: 9/9 (EXPL-01 through EXPL-04, INSP-01 through INSP-03, SRCH-01 through SRCH-02)

---

### ✅ Phase 3: Scoped Graph Core
**Goal**: User can understand the active workspace or selected folder through a scoped interactive graph.

**Success Criteria**:
1. User can switch to Graph Mode and see folders/files from the active scope.
2. Containment relationships render as graph edges.
3. Clicking a node updates the same inspector used by Explorer Mode.
4. Double-clicking a node opens a file or focuses a folder.
5. Oversized scopes are capped and represented with cluster/summary nodes.
6. Graph requests stay explicitly scoped by active root path and graph mode.

**Plans Completed**:
- ✅ 03-01: Graph API, visible-scope query, and Graphology data loading
- ✅ 03-02: Sigma.js rendering, styling, pan/zoom, and node interactions
- ✅ 03-03: Node caps, folder clustering, and graph-to-inspector/open integration

**Requirements**: 5/5 (GRPH-01 through GRPH-05)

---

### ✅ Phase 4: Performance Guardrails
**Goal**: Orbit stays responsive and reuses cached metadata when working with large local folders.

**Success Criteria**:
1. Explorer, graph, and previews lazy-load rather than pushing entire workspaces into the UI.
2. Scans run in background Rust tasks without freezing the React interface.
3. Closing and reopening the app reuses SQLite metadata from prior scans.
4. Large folders render as summaries or clusters instead of thousands of visible nodes.

**Plans Completed**:
- ✅ 04-01: Lazy loading, background scan behavior, and cache reuse validation
- ✅ 04-02: Large-folder graph limits, cluster summaries, and responsiveness checks

**Requirements**: 3/3 (PERF-01 through PERF-03)

## Architecture Delivered

**Stack**:
- Desktop shell: Tauri 2
- Native engine: Rust
- UI: React + TypeScript
- Persistent database: SQLite with WAL mode
- Graph model: Graphology (in-memory)
- Graph renderer: Sigma.js
- Search: SQLite FTS5
- File watching: Rust notify crate

**Backend (Rust)**:
- File scanning with metadata extraction
- Safe file operations
- Database writes with transaction safety
- Filesystem watching
- Graph scope enforcement
- Search queries
- Opening files/folders externally

**Frontend (React/TypeScript)**:
- Layout orchestration
- Graph rendering
- Explorer panels (tree, list, search)
- Inspector panel
- Mode switching
- Virtual scrolling for large lists
- Performance monitoring

## Key Features Delivered

### Workspace Management
- Open folder as active workspace
- Visual indicator of current workspace
- Manual scan trigger

### File Indexing
- SQLite persistence with WAL mode
- Comprehensive metadata (path, name, parent, extension, MIME type, size, timestamps, hash)
- Incremental scanning (skip unchanged files)
- Progress reporting with UI feedback
- Durable logging for diagnostics

### Explorer Views
- Tree view with lazy loading for large directories
- List view with parent navigation
- Selection state shared across views
- Virtual scrolling for lists >50 items

### Inspector Panel
- Full metadata display
- Available actions (open externally, copy path, show in folder)
- Basic preview for text and image files
- Lazy preview loading with loading states

### Search
- Filename search with debouncing (300ms)
- Result selection and navigation
- Integration with inspector and external open

### Graph Mode
- Sigma.js rendering with color-coded nodes
- Pan/zoom controls with level display
- 200 node cap for performance
- Folder clustering for directories with >50 children
- Click to inspect
- Double-click to open files or focus folders
- Cluster expansion (double-click shows more items)

### Performance Safeguards
- Virtual scrolling for explorer lists
- Cache validation on app startup
- Performance monitoring with FPS tracking
- Lazy loading throughout UI
- Background scan verification (non-blocking)
- Responsiveness warnings
